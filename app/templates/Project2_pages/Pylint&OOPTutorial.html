OOP & Pylint Tutorial

OOP & 4 Pillars

OOP is a computer programming paradigm where you create objects to work with.

The four principles of OOP are Encapsulation, Abstraction, Inheritance, and Polymorphism.

Pylint

Pylint is a tool created to inspect python code for errors and give feedback on how to improve the code to meet PEP8 style guide. PEP8 style guide/rulebook is document that helps python coders write neat and well designed code. It was written by Guido Van Rossum, Barry Warsaw, and Nick Coghlan.

Install Pylint

Windows
pip install pylint
MacOS
pip install pylint
Linux
sudo apt-get install pylint

OOP Glossary

Factory Pattern: A factory is used to generate instances of other classes.

Instantiation: The process of creating an object of a class. This is done with the constructor method __init__(self).

Object:An instance of a class.

Class:A template or blueprint from which objects can be instantiated from.

Namespace:A collection of defined symbolic names along with information about the object that each name references.

Constructor:A method of a class that is automatically called when an object is instantiated from its class.

Fixture:Fixed testing environment in which test are performed so that results are repeatable.

Type Hint:Gives you the ability to define the type of values that can be passed for each argument of a function or method. Type

Type Cast:Assigning a value of one primitive data type to another type.

Unit Test:Creating test cases to verify all possible states and configuration of a code in order to discover bugs.

Static:The static type of an object is the declared type of the variable used to refer to it. Restricted in what data they can access.

Class Method:Methods which are called on a class instead of an instance.

Facade:A structural design pattern that provides a simpler unified interface to a more complex system.

Design Pattern:a general repeatable solution to a commonly occurring problem in software design

Method:A subroutine defined within a class to implement a behaviour.

property:A variable defined within a class to store a value.

General Tips

Be Consistent With Variables
Example: full_name, first_name, last_name Avoid: a_name, b_name, x, y
Limit Using Too Long/Short Names & Numbers
Example: test_calculator_multiply_staticAvoid: function_to_test_the_addition_calculator, x = "My name",1_name, first_name_1
Use Detailed Variable Names
Example: add_numbers = "Addition Calculator"Avoid: a = "Addition Calculator"
Follow The Lowercase With Underscore Naming Convention
Example: add_numbers, multiply_number, subtract_numbers
Documentation String (Used to describe functions, classes and methods)
